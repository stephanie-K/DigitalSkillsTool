We are using the GOV.UK Prototype Kit {{releaseVersion}}

{{releaseVersion | log }}

This kit lets you rapidly create HTML prototypes of GOV.UK services.

It contains all you need to prototype anything from simple static pages to complex, data-driven transactions.

In the sections below, you will find some information about how we have used it to suits our needs. Some of this information is for developers mostly, but not all.

If you have any question, please get in touch.

  • GOV.UK Design System

How we are using it

Different forms are needed to support the classes. Some are Council forms, some are GOVUK forms or Scottish Government forms and some are from private services like energy providers.

Different layouts have been created for each and even among each category, there is a variety of layouts:
For example, the DVLA have a different layout and style compared to HMRC. Social Security Scotland also has its own style.

Important files and paths

Our code is in this Github repository

Layout files in app>views>

  • layout.html - default GOV.UK layout
  • layout_erc.html - customised for Council forms
  • layout_sg.html - customised for Scottish Government
  • layout_sss.html - customised for Social Security Scotland
  • layout_tool.html - look like the council one but for high level pages of the tool, like the one you're reading just now

CSS/SASS files in app>assets>sass>

  • application.scss - default GOV.UK plus some css we added for GOVUK forms
  • erc.scss - styles to support the council forms
  • sg.scss - styles to support the Social Security Scotland and Scottish Government forms
  • other.scss - styles for other forms

Each form has a folder app>views>forms>

All images are in app>assets>images>

In your code, the path will be: {{ asset_path }}images/name-of-your-image-file

Both the images and the styles are copied in the public folder. You should not put anything there, it's done automatically for you.

Route.js app>

This file is key to introduce navigation logic, persistance of data and validation to the forms. It is shared by all the forms.

Easy update of dates or other variables

Server.js at the route of the code.

This file is where we can add variables that are available in all views. It is shared by all the forms.

We are using it to easily update various dates used by forms like Bulb, Bidding on properties, the journal of the Universal Credit, booking a theory test appointment date.
